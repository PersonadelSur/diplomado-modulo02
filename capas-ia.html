Las capas que determinan la calidad del output
No es la herramienta. Es cómo la usas.
Dos PMs con la misma herramienta obtienen resultados radicalmente distintos. La diferencia no está en el modelo — está en las capas de trabajo que cada uno aporta. Estas son las 5 capas que debes entender ahora, aunque profundizarás en ellas en el Módulo 6.
Capa 1
Prompt
La instrucción
La forma en que formulas la pregunta determina el tipo de respuesta. Un prompt vago genera outputs genéricos. Un prompt con rol, restricciones y formato esperado genera outputs accionables.
"Dame ideas de producto" → genérico · "Eres PM de una app de transporte. Dame 10 hipótesis de abandono del flujo de pago, ordenadas de más a menos frecuente, en formato lista numerada" → útil
El PM lo aplica cuando…
Especifica el rol, el contexto del producto, el formato esperado y las restricciones antes de hacer la pregunta.

Capa 2
Context
La información
La IA no sabe nada de tu producto si no se lo dices. Subir archivos, pegar transcripciones, adjuntar datos reales transforma un chat genérico en un analista que conoce tu contexto.
Subir 5 transcripciones de entrevistas + pedir síntesis de patrones → la IA trabaja con evidencia real, no suposiciones genéricas de industria
El PM lo aplica cuando…
Carga documentos de research, datos de uso, feedback de usuarios y reportes antes de hacer cualquier análisis con IA.
NotebookLM Claude (archivos) Gemini Gems
Capa 3
Intent
El objetivo
¿Para qué vas a usar el output? ¿Para presentar a stakeholders, para guiar una entrevista, para priorizar el backlog? El intent cambia el tono, el nivel de detalle y el formato óptimo.
"Quiero este análisis para presentarlo a un comité ejecutivo en 5 minutos" → la IA ajusta densidad, jerarquía visual y lenguaje ejecutivo automáticamente
El PM lo aplica cuando…
Declara explícitamente la audiencia y el uso del output: "esto es para el equipo de dev", "esto es para el CEO", "esto es para guía interna".
Gamma Claude ChatGPT
Capa 4
Especificación
El formato
Definir el formato de salida — tabla, lista numerada, JSON, párrafo ejecutivo, bullet points — elimina ambigüedad y reduce el trabajo de post-procesamiento. Especificar es eficiencia.
"Responde en una tabla de 3 columnas: Insight | Evidencia | Acción recomendada. Máximo 6 filas." → output listo para pegar en Notion o presentación
El PM lo aplica cuando…
Siempre especifica longitud, estructura y formato antes de enviar el prompt — ahorra tiempo de edición post-generación.

Capa 5
Conexión
La integración
El mayor salto de productividad ocurre cuando las herramientas se conectan entre sí: Granola transcribe → Claude sintetiza → Gamma presenta → Notion almacena. El PM orquesta el flujo.
Entrevista grabada → Granola transcribe → Claude extrae insights → NotebookLM almacena → n8n envía resumen al canal de Slack del equipo automáticamente
El PM lo aplica cuando…
Diseña el pipeline de datos del equipo: define qué entra, qué herramienta procesa, qué sale y dónde se almacena. Se profundiza en Módulo 6.

⚡
Principio clave: la IA es tan útil como la información que le das
Un PM que trabaja con contexto rico (archivos reales, datos de usuarios, documentos del producto) obtiene outputs 10x más útiles que uno que solo pregunta desde cero. El trabajo previo de recolección de información es parte del trabajo de IA. Cargar bien el contexto no es opcional — es la diferencia entre un asistente genérico y un analista que entiende tu producto.
Automatización — lo que viene en módulos 5 y 6
Cuando las herramientas se conectan solas
La automatización es la capa que multiplica el impacto. En lugar de hacer cada paso manualmente, el PM diseña flujos que conectan herramientas y ejecutan tareas repetitivas automáticamente. Aquí una preview de lo que construirás más adelante.
n8n
Open Source · Autoalojado o Cloud
Automatización de flujos visuales con nodos conectables. Permite integrar cualquier herramienta con API — incluyendo Claude, Notion, Google Sheets, Slack y más — sin código.
Flujo ejemplo: Typeform (feedback) → n8n → Claude API (síntesis) → Notion (almacena insight) → Slack (notifica al equipo)
Make (antes Integromat)
SaaS · Más accesible para no-técnicos
Similar a n8n pero con interfaz más visual y menor curva de aprendizaje. Ideal para equipos sin perfil técnico que quieren automatizar sin instalar nada.
Flujo ejemplo: Google Forms (entrevista) → Make → ChatGPT (extrae citas) → Google Sheets (registra) → Gmail (avisa al investigador)
Google Workspace Studio
Gratuito · Integrado en Workspace
IA de Gemini integrada directamente en Sheets, Docs y Slides. Si el equipo ya usa Google Workspace, es el punto de entrada más rápido a la automatización con IA sin cambiar de herramientas.
Flujo ejemplo: Datos de NPS en Sheets → Gemini analiza → genera resumen en Docs → crea slide de hallazgos en Slides automáticamente
